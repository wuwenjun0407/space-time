## 1. 架构总览

```
┌────────────────────────────────────────────────┐
│                   用户设备                       │
│  ┌───────────────┐     ┌────────────────────┐  │
│  │  Flutter APP  │     │  Next.js 后台管理   │  │
│  └───────┬───────┘     └──────────┬─────────┘  │
└──────────┼────────────────────────┼─────────────┘
           │ HTTP                   │ HTTP
           ▼                        ▼
┌─────────────────────────────────────────────────┐
│  Nginx (80)  反代 → FastAPI(8000) / Next.js(3000) │
└──────────────────────────┬──────────────────────┘
                           │
                           ▼
              ┌───────────────────────┐
              │   FastAPI (8000)      │
              │   SQLAlchemy Async    │
              │   Alembic Migration   │
              └───────────┬───────────┘
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
┌──────────────┐  ┌──────────────┐  ┌───────────┐
│ PostgreSQL   │  │   七牛云      │  │   SMTP    │
│ (5432)       │  │ family-media │  │  (可选)   │
└──────────────┘  └──────────────┘  └───────────┘
```

**核心决策**：

- 无 Redis：验证码用 DB 表 + TTL 过期；
- 无 CDN：七牛签名外链直出；
- 无域名 / 无 HTTPS / 无备案 / 无短信 / 无推送；
- 群组即内容池：文件归属群组（`files.group_id`），空间通过 `space_files` 引用；
- 上传独立副本：多群组上传时每群组各存一份 file 记录 + 七牛对象；
- 双端上传：APP 和 Admin 后台共享 `POST /files/upload-token` API，逻辑一致。

## 5. 后端核心设计

### 5.1 应用层结构

```
Request → Middleware(CORS/RateLimit/ErrorHandler)
       → Router → Dependency(Auth/DB Session)
       → Service → Repository/ORM → DB
       → Response(Pydantic Schema)
```

### 5.2 关键设计决策

1. **全异步**：asyncpg + async session；
2. **分层**：Router → Service → ORM Model；
3. **权限**：依赖注入 `get_current_user` + `require_role()` + `require_group_owner()`；
4. **分页**：统一 `page/page_size` + `PaginatedResponse`；
5. **错误处理**：自定义 `AppException` → 全局 handler → 统一 JSON；
6. **限流**：slowapi 基于 IP；
7. **Admin 路由前缀**：`/api/v1/admin/...`；
8. **后台权限分层**：路由依赖注入中根据 `group_members.role` 判断；
9. **上传 API 共用**：`POST /files/upload-token` 由 APP 和 Admin 共同调用，仅校验 JWT 有效性 + 群组成员身份。

## 6. 数据库设计（PostgreSQL DDL）

### 6.1 Schema

```sql
CREATE SCHEMA IF NOT EXISTS spacetime_db;
SET search_path TO spacetime_db;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

### 6.2 完整 DDL

```sql
-- ============================================================
-- 用户
-- ============================================================
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    username VARCHAR(20) NOT NULL UNIQUE,
    nickname VARCHAR(50),
    avatar_url TEXT,
    avatar_audit_state VARCHAR(20) DEFAULT 'none',
    password_hash VARCHAR(200) NOT NULL,
    role VARCHAR(20) NOT NULL DEFAULT 'user',       -- superadmin / user
    status VARCHAR(20) NOT NULL DEFAULT 'active',   -- active / banned / pending_delete
    failed_login_count INT DEFAULT 0,
    last_failed_at TIMESTAMPTZ,
    delete_requested_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================================
-- 群组
-- ============================================================
CREATE TABLE groups (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(100) NOT NULL,
    description TEXT,
    owner_id UUID NOT NULL REFERENCES users(id),
    is_personal BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE group_members (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    group_id UUID NOT NULL REFERENCES groups(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL DEFAULT 'member',     -- owner / member
    is_active BOOLEAN DEFAULT FALSE,
    joined_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(group_id, user_id)
);

-- 每用户同时只有一个 is_active=true 的群组
CREATE UNIQUE INDEX idx_one_active_group
    ON group_members(user_id) WHERE is_active = TRUE;

-- ============================================================
-- 分类
-- ============================================================
CREATE TABLE categories (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    group_id UUID NOT NULL REFERENCES groups(id) ON DELETE CASCADE,
    name VARCHAR(50) NOT NULL,
    sort_order INT DEFAULT 0,
    is_default BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(group_id, name)
);

-- ============================================================
-- 文件（群组内容池）
-- ============================================================
CREATE TABLE files (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    group_id UUID NOT NULL REFERENCES groups(id),
    category_id UUID REFERENCES categories(id) ON DELETE SET NULL,
    uploader_id UUID NOT NULL REFERENCES users(id),
    file_type VARCHAR(10) NOT NULL,          -- image / video
    mime_type VARCHAR(50),
    original_filename VARCHAR(255),
    storage_key VARCHAR(500) NOT NULL,
    file_size BIGINT,
    width INT,
    height INT,
    duration_seconds FLOAT,
    description TEXT,
    shot_at TIMESTAMPTZ,
    upload_source VARCHAR(10) DEFAULT 'app', -- app / admin（标记上传来源）
    status VARCHAR(20) DEFAULT 'draft',      -- draft/uploading/pending_callback/ready/failed
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_files_group_shot ON files(group_id, shot_at DESC);
CREATE INDEX idx_files_uploader ON files(uploader_id);
CREATE INDEX idx_files_category ON files(category_id);

-- ============================================================
-- 空间
-- ============================================================
CREATE TABLE spaces (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    group_id UUID NOT NULL REFERENCES groups(id) ON DELETE CASCADE,
    creator_id UUID NOT NULL REFERENCES users(id),
    name VARCHAR(100) NOT NULL,
    description TEXT,
    cover_url TEXT,
    visibility VARCHAR(20) DEFAULT 'public_in_group', -- public_in_group / members / private
    is_default BOOLEAN DEFAULT FALSE,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 每人每群组只有一个 is_default=true
CREATE UNIQUE INDEX idx_one_default_space
    ON spaces(group_id, creator_id) WHERE is_default = TRUE;

-- 空间可见成员（visibility=members 时）
CREATE TABLE space_members (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    space_id UUID NOT NULL REFERENCES spaces(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    added_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(space_id, user_id)
);

-- 空间文件（策展引用表）
CREATE TABLE space_files (
    space_id UUID NOT NULL REFERENCES spaces(id) ON DELETE CASCADE,
    file_id UUID NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    sort_order INT DEFAULT 0,
    added_by UUID REFERENCES users(id),
    added_at TIMESTAMPTZ DEFAULT NOW(),
    PRIMARY KEY (space_id, file_id)
);

CREATE INDEX idx_space_files_sort ON space_files(space_id, sort_order);

-- 空间皮肤（直接引用主题）
CREATE TABLE space_themes (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    space_id UUID NOT NULL REFERENCES spaces(id) ON DELETE CASCADE UNIQUE,
    theme_id UUID REFERENCES themes(id) ON DELETE SET NULL,
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================================
-- 互动
-- ============================================================
CREATE TABLE likes (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    file_id UUID NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(file_id, user_id)
);

CREATE TABLE favorites (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    file_id UUID NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    hidden BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(file_id, user_id)
);

CREATE TABLE comments (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    file_id UUID NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    reply_to UUID REFERENCES comments(id) ON DELETE SET NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ============================================================
-- 视觉素材 & 皮肤体系（两层：风格 → 主题氛围包）
-- ============================================================
CREATE TABLE styles (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    key VARCHAR(50) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    sort_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 主题 = 完整视觉氛围包（背景+动效+BGM+字体+色彩模式）
CREATE TABLE themes (
```

### 6.3 关键设计说明

1. **users.role**：只有两档 `superadmin / user`，群组内的角色用 `group_members.role` 区分；
2. **group_members.role**：`owner / member`，owner = 主理人，member = 普通成员；
3. **spaces.visibility**：三档权限，`members` 时走 `space_members` 表；
4. **spaces.is_default**：部分唯一索引确保每人每群组最多一个默认空间；
5. **space_files**：策展引用表，文件删除 CASCADE 级联清除引用；
6. **上传独立副本**：多群组上传时每个群组各自一条 file 记录 + 独立七牛对象；
7. **files.upload_source**：标记文件从 APP 还是 Admin 上传（仅统计用，不影响权限）；
8. **themes 两层设计**：themes 表直接内嵌背景key/动效配置JSON/BGM key/字体/color_mode，无需拆分为 effects + audios + presets 三张额外表；
9. **themes.color_mode**：`dark`（深色主题如科技/暗夜，浅色文字）/ `light`（浅色主题如极简白/乡村，深色文字）；
10. **themes.effect_config**：空对象 `{}` 表示无动效叠加，有值时包含 type/count/colors 等粒子参数；
11. **user_active_theme**：替代旧 user_preset_apply，记录用户当前激活的主题；
12. **space_themes**：替代旧 space_skins，每个空间通过 theme_id 引用一个主题；
13. **去掉 invites 表**：不再需要邀请码，主理人直接按用户名添加。

```dart
class AppThemeNotifier extends StateNotifier<ThemeModel?> {
  void applyTheme(ThemeModel theme) {
    state = theme;
    // 背景切换：bg_image_key 或 bg_video_key 生成七牛签名URL
    ref.read(backgroundProvider.notifier).update(theme);
    // 动效：effect_config 非空则启动粒子，空则关闭
    if (theme.effectConfig.isNotEmpty) {
      ref.read(particleConfigProvider.notifier).update(theme.effectConfig);
    } else {
      ref.read(particleConfigProvider.notifier).disable();

    // BGM：bgm_key 非空则播放

    ref.read(fontProvider.notifier).update(theme.fontDisplay, theme.fontBody);
    // 色彩模式：color_mode 决定文字和卡片亮暗
    ref.read(colorModeProvider.notifier).update(theme.colorMode);
  }
}
```

```dart
class BgmService {
  final AudioPlayer _player = AudioPlayer();

  Future<void> play(String? url, {double volume = 0.3}) async {
    if (url == null) { await stop(); return; }
    await _player.setUrl(url);
    await _player.setVolume(volume);
    await _player.setLoopMode(LoopMode.all);
    await _player.play();
  }

  Future<void> stop() async {
    for (var v = _player.volume; v > 0; v -= 0.05) {
      await _player.setVolume(v);
      await Future.delayed(Duration(milliseconds: 50));
    }
    await _player.stop();
  }
}
```

```tsx
                style={{ background: theme.colorMode === 'dark' ? 'rgba(255,255,255,0.08)' : 'rgba(0,0,0,0.06)' }}
            >
                <h3 style={{ color: theme.colorMode === 'dark' ? '#fff' : '#1A1A2E' }}>{theme.name}</h3>
                <p style={{ color: theme.colorMode === 'dark' ? 'rgba(255,255,255,0.6)' : 'rgba(0,0,0,0.5)' }}>预览效果</p>
            </div>
            {/* BGM 播放按钮 */}
            {theme.bgmKey && <BgmPlayer src={signQiniuUrl(theme.bgmKey)} volume={theme.bgmVolume} />}
        </div>
```
