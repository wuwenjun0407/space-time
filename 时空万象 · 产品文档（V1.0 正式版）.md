  6. **管理平台人人可用**：所有注册用户可登录后台，普通成员只能管理自己的空间 + 上传；
  7. **首页皮肤体系**：后台按风格配置多套主题（背景图/视频 + 动效 + BGM + 字体），系统预置 8 风格 16 主题，APP 时光轴一键换肤；
  8. **5 Tab**：时光（沉浸展示 + 一键换肤）| 空间 | 上传 | 回忆（搜索/探索/情感）| 我的。

## 1. 产品总览与目标用户

### 1.1 产品定位

「时空万象」是面向**个人 / 家庭 / 小型亲友团**的私密影像存储、记录、互动与沉浸式展示产品。以**群组**为数据隔离与内容池单元，以**空间**为策展展示载体，以**风格 - 主题**为视觉与氛围核心。

### 1.2 三类目标用户

| 用户画像   | 典型场景                     | 核心诉求                     |
| ---------- | ---------------------------- | ---------------------------- |
| 个人记录者 | 记录成长、旅行、宠物、作品   | 私密、易搜索、按时间线浏览   |
| 家庭主理人 | 家庭照片、孩子成长、长辈纪念 | 多成员共享、权限可控、仪式感 |
| 亲友小圈子 | 同学聚会、好友旅行、社群作品 | 群组隔离、互动、氛围沉浸     |

### 1.3 两端分工

| 职能域                     | APP 端 | Admin 后台管理平台 |
| -------------------------- | ------ | ------------------ |
| 浏览群组内容（时光轴）     | ✅     | —                  |
| 上传照片/视频到群组        | ✅     | ✅（所有用户）     |
| 互动（点赞/评论/收藏）     | ✅     | 查看/删除违规      |
| 查看空间                   | ✅     | —                  |
| 回忆（搜索/筛选/情感）     | ✅     | —                  |
| 消息通知                   | ✅     | —                  |
| 群组切换                   | ✅     | ✅                 |
| 首页皮肤一键切换           | ✅     | —                  |
| 皮肤配置与管理             | —      | ✅（超管/主理人）  |
| 创建/编辑/装扮/删除空间    | ❌     | ✅（本人空间）     |
| 空间策展（选文件加入空间） | ❌     | ✅（本人空间）     |
| 空间皮肤配置               | ❌     | ✅（本人空间）     |
| 群组成员管理（添加/移除）  | ❌     | ✅（主理人）       |
| 文件批量管理/删除          | ❌     | ✅（主理人）       |
| 分类管理                   | ❌     | ✅（主理人）       |
| 用户封禁/注销              | ❌     | ✅（超管）         |
| 操作日志/系统设置          | ❌     | ✅（超管）         |

> **一句话**：APP = 上传 + 浏览 + 互动 + 一键换肤；后台 = 上传 + 创建空间 + 装扮 + 策展 + 皮肤配置 + 管理。

### 1.4 核心能力清单

1. 一人一群组 + 可被邀请加入多个群组
2. JWT 登录（用户名 + 密码 + 图形验证码）
3. **群组即内容池**：成员上传内容汇入当前群组
4. **上传选群组**：APP 和后台均可上传，可同时上传到多个群组（独立副本）
5. **空间策展**：每人在后台管理自己的空间，从群组内容中选文件展示
6. **空间权限三档**：群组公开 / 指定成员 / 私有
7. 七牛云直传 + 回调入库
8. 完整互动（点赞 / 评论 / 收藏）
9. **首页皮肤体系**：后台按风格配置多套主题（背景图/视频 + 动效叠加 + BGM + 字体），系统预置 8 风格 16 主题，APP 一键切换
10. 沉浸式渲染（背景图/视频 / 粒子动效叠加 / BGM）
11. 消息中心、回忆模块、操作日志

## 2. 全局名词与核心规则

### 2.1 名词定义表

| 名词             | 定义                                                                     |
| ---------------- | ------------------------------------------------------------------------ |
| 群组（Group）    | 数据隔离与内容池单元；一个群组 = 一个共享相册池                          |
| 我的群组         | 注册时自动创建，用户是主理人，不可删除                                   |
| 受邀群组         | 被他人添加加入的群组，可主动退出                                         |
| 生效群组         | 同一时刻仅一个群组生效，APP 全部数据源跟随它                             |
| 空间（Space）    | 策展集合 — 从群组内容中引用文件的主题展览；有独立视觉风格                |
| 默认空间         | 用户在某群组中指定的一个空间，空间 Tab 进入时首先展示                    |
| 分类（Category） | 文件归档目录，群组内主理人管理                                           |
| 风格（Style）    | 视觉与氛围大类（科技 / 乡村 / 纯欲 …）                                   |
| 主题（Theme）    | 风格下一套完整视觉氛围包：背景图/视频 + 动效叠加 + BGM + 字体 + 色彩模式 |
| 首页皮肤         | 用户在 APP 时光轴激活的主题，决定首页视觉氛围                            |
| 主理人（Owner）  | 群组创建者，拥有最高管理权限                                             |
| 策展（Curation） | 在后台从群组内容中选取文件加入空间、排序                                 |

### 2.2 账号与身份

1. 注册方式：用户名 + 密码 + 图形验证码（不接短信）；
   - 用户名：4~20 字符，字母/数字/下划线/中文，全局唯一，注册后不可修改；
   - 密码：8~32 字符，必须含字母与数字；
   - 验证码：4 位字母+数字，TTL 2 分钟，提交后即失效；
   - 同一 IP 注册 ≤ 3 次/分钟、≤ 20 次/日；
2. 登录方式：用户名 + 密码 + 图形验证码（连续失败 ≥ 3 次后强制）；
3. APP 与后台共享同一 JWT 登录态；
4. 注册成功后系统自动：
   - 分配默认头像（系统头像库随机）；
   - 生成「我的群组」并设为生效；
   - 在我的群组下生成默认分类；
   - 写入默认首页皮肤 = 科技风「银河漫游」主题；
5. 头像规则：系统默认 / 用户自定义（≤ 5MB，走审核）；
6. 修改约束：用户名不可改；密码可在 APP 自助改（旧密码 + 验证码）；忘记密码由超管后台重置。

### 2.3 群组核心规则

1. **一人一群组**：注册时自动创建「我的群组」，用户是主理人；
2. 同一时刻**仅一个生效群组**；切换时原群组失效；
3. 硬兜底：退出受邀群组导致生效消失 → 自动回退我的群组；
4. 我的群组：不可退出、不可删除、不可转让；
5. 受邀群组：可主动退出；主理人可将你移除；
6. 上传时选择目标群组（可多选），文件以独立副本形式存入各群组，每份独立归属对应群组；
7. 群组内所有成员的上传文件汇集 = 该群组的全部内容。

### 2.4 成员添加规则（替代旧邀请码机制）

1. 仅群组主理人可在后台添加成员；
2. 添加方式：按用户名搜索 → 选择角色 → 确认加入；
3. 被添加者必须已完成注册；
4. 被添加后，对方 APP 消息中心收到通知，自动出现在群组列表中（无需「接受」步骤）；
5. 主理人可随时移除成员；被移除者数据归属不变（已上传的文件保留在群组中）。

### 2.5 空间规则

1. 群组内每个成员都可以创建空间（在后台管理平台）；
2. **创建 + 装扮 + 策展**都在管理平台完成；
3. APP 端只负责浏览空间、互动；
4. 空间不拥有文件，只持有引用 + 排序（`space_files`）；
5. 同一文件可出现在多个空间中；
6. 群组内容中删除文件 → 所有引用该文件的空间自动失去该条目；
7. **空间访问权限三档**：
   - 群组公开：群组内所有成员可见；
   - 指定成员：创建者手动选择谁可见；
   - 私有：仅创建者可见；
8. 对他人空间：仅浏览 + 互动（赞/评/藏），不可编辑/删除/策展；
9. **默认空间**：用户在后台将自己的某个空间标记为默认，APP 空间 Tab 进入时优先展示它。

### 2.6 分类与上传规则

1. 上传时可选分类（从目标群组的分类列表中选）；未选归入「默认分类」；
2. 分类由群组主理人在后台管理，普通成员可见但不可管理；
3. 上传时选择目标群组（可多选，独立副本）；
4. **上传入口**：APP Tab3 + 后台「影像上传」模块（两端共享同一上传 API）；
5. APP 无文件删除入口；后台主理人/文件上传者可删除；
6. 上传强校验：
   - 图片：JPG / PNG / WebP / HEIC，≤ 50MB；
   - 视频：MP4 / MOV，≤ 1GB，≤ 30min；
   - 单次批量 ≤ 50 文件；
   - 客户端先校验格式/大小，再申请上传 token。

### 2.7 权限矩阵（作用域 = 当前群组）

| 角色       | 上传（APP+后台） | 浏览群组内容 | 浏览空间     | 互动 | 后台管理                                         |
| ---------- | ---------------- | ------------ | ------------ | ---- | ------------------------------------------------ |
| 平台超管   | ✅               | 全量         | 全量         | 全开 | 全部                                             |
| 群组主理人 | ✅               | 全量         | 全量         | 全开 | 成员管理 + 分类 + 所有空间 + 文件管理 + 皮肤配置 |
| 普通成员   | ✅               | 全量         | 有权限的空间 | 全开 | 仅自己的空间（创建/装扮/策展）+ 上传             |

### 2.8 互动 & 通知规则

- 图片/视频均支持点赞、评论、收藏；
- 点赞/评论推送通知给**上传者**；
- 收藏内容失去可见权限：自动隐藏 + 弹窗提示；权限恢复后自动展示；
- 评论支持单层回复（@用户），首版不做多级嵌套。

### 2.9 首页时光轴规则

- 数据范围：当前生效群组中当前用户有权查看的全部文件；
- 排序：拍摄时间 DESC，缺失则上传时间兜底；
- 加载：分页懒加载，单页 ≤ 30 条，预拉取下一页。

## 3. 视觉系统总纲（风格 · 主题 · 首页皮肤）✨

### 3.1 设计目标

- 默认沉浸式科技感、星空未来感；
- 用户可在 APP 一键切换首页皮肤；
- **皮肤 = 主题**：主题一键应用，同时切换背景图/视频 + 动效叠加 + BGM + 字体；
- **皮肤在后台配置**：超管/主理人在后台管理平台的「素材主题管理」中创建和配置每个主题的背景、动效参数、BGM 及字体等全部参数；
- **系统预置主题**：首批 8 套风格共 16 个系统预置主题，开箱即用；
- 仅切换「皮肤层」，不影响业务数据。

### 3.2 两层体系

```
风格（Style）
 └─ 主题（Theme）= 一套完整视觉氛围包
      ├─ 背景图片 / 背景视频        主视觉，必填其一
      ├─ 动效叠加（可选）           粒子 / 流光等，effect_config JSON
      ├─ BGM 背景音乐（可选）       七牛 storage_key，含循环/音量参数
      ├─ 字体                       标题字体 + 正文字体
      └─ color_mode: dark | light   决定文字/卡片色调
```

### 3.3 皮肤配置与应用的完整闭环

```
┌─────────────────────────────────────────────────────────────┐
│  Admin 后台（素材主题管理）                                    │
│  超管/主理人操作：                                            │
│  1. 创建风格 → 2. 在风格下创建主题                            │
│  3. 配置主题：上传背景图/视频 + 设置动效参数 + 上传BGM + 字体  │
│  4. 设置主题状态（上架/下架）+ 预览效果                        │
└──────────────────────────┬──────────────────────────────────┘
                           │ API: GET /styles, /themes
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  APP 端（首页时光轴）                                         │
│  用户操作：                                                   │
│  顶部右侧「皮肤」按钮 → 弹出皮肤选择面板 →                    │
│  浏览风格列表 → 选择主题 → 预览效果 → 一键应用                 │
│  写入 user_active_theme → 首页热切换背景/动效/BGM/字体         │
└─────────────────────────────────────────────────────────────┘
```

> **以上 16 个主题在系统初始化时 seed 入库（`is_system = true`），用户注册即可直接使用。**

### 3.5 一键换肤规则（APP 首页）

- **应用对象**：APP 时光轴首页皮肤（本人操作）；
- **入口**：时光 Tab 顶部右侧「皮肤」按钮 → 弹出皮肤选择面板；
- **面板内容**：风格分类横向 Tab → 该风格下可用主题列表 → 点击预览 → 确认应用；
- **保存**：写入 `user_active_theme`（target_type = 'home'）；
- **即时生效**：应用后首页背景、动效、BGM、字体热切换，无需重启；
- **撤销**：60s 内可撤销回上一个主题；
- **还原**：始终保留"恢复系统默认"选项（= 科技风「银河漫游」）。

### 3.6 空间主题（独立于首页皮肤）

- 空间主题由空间创建者在后台配置；
- 每个空间有独立主题配置（`space_themes` 表，引用 `theme_id`）；
- 进入空间时覆盖首页皮肤，退出空间恢复首页皮肤。

### 3.7 性能约束

- 帧率：中端 ≥ 50fps、低端 ≥ 30fps；
- 动效分级：high / mid / low / off（设备评分自动）；
- 全局动效开关 + 系统 Reduce Motion 自动关闭；
- BGM 默认静音启动。

## 4. APP 移动端 — 信息架构与页面规范

### 4.1 信息架构（5 Tab）

```
APP
├─ 启动页 / 引导页
├─ 登录 / 注册
├─ 主框架（5 Tab）
│   ├─ Tab1 时光（沉浸式影像展示 + 一键换肤）
│   ├─ Tab2 空间（有权限的空间列表 + 浏览）
│   ├─ Tab3 上传（中央放大按钮）
│   ├─ Tab4 回忆（搜索 / 筛选 / 时间段 / 分类 / 情感模块）
│   └─ Tab5 我的
└─ 二级页：文件详情 / 互动列表 / 消息中心 / 收藏 /
          个性化中心 / 群组切换 / 偏好设置 / 空间内容浏览
```

### 4.2 全局 UI 规范

- 颜色模式：根据当前激活主题的 color_mode（dark/light）决定文字与卡片色调；
- 字体：Inter / 思源黑体；
- 圆角：8 / 12 / 20；卡片 16；
- 触控热区：≥ 44pt；
- 暗色模式默认开，亮色可选；
- 首版仅简体中文。

### 4.3 启动流程

1. 启动页 → 鉴权检测 → 未登录跳登录 / 已登录刷 token → 拉首屏数据 + 用户当前皮肤配置；
2. 强制升级判定（`/system/version`）；
3. 首次启动隐私政策弹窗。

## 5. APP 移动端 — 各 Tab 详细设计

### 5.1 Tab1 时光（沉浸式影像展示 + 一键换肤）

> **设计理念**：纯视觉沉浸体验，无搜索框、无筛选条，打开即是全屏影像流。搜索/探索功能由 Tab4「回忆」承担。首页视觉氛围由当前激活主题控制。

#### 5.1.1 视觉

- 时间轴：左侧时间柱（年/月），右侧卡片瀑布；
- 字体及卡片文字色调受当前主题 color_mode 控制（dark=浅色文字，light=深色文字）。

#### 5.1.2 一键换肤交互

1. 点击右上角「皮肤」图标；
2. 弹出底部半屏面板（BottomSheet）；
3. 顶部：风格分类 Tab 横向滚动（科技 / 乡村 / 纯欲 …）；

7. 底部 Toast：「已应用 XX 皮肤，60s 内可撤销」。

#### 5.1.3 数据源

当前生效群组中**全部文件**，按拍摄时间 DESC。

#### 5.1.4 交互

- 下拉刷新；触底加载；双指捏合切换密度（年/月/日）；
- 长按弹快捷操作（收藏 / 详情）；
- 点击进入文件详情；
- **无搜索 / 无筛选 UI** — 纯浏览体验。

#### 5.1.5 文件详情页

- 大图 / 视频播放；

- 互动条：点赞 / 评论 / 收藏；
- 「查看所有互动」BottomSheet；
- 左右滑切换相邻文件。

#### 5.1.6 异常

- 无内容：空状态插画 + 引导上传；

### 5.2 Tab2 空间

#### 5.2.1 列表

- 我的默认空间（置顶高亮）；
- 我的其他空间；
- 他人有权限的空间；
- 卡片：封面 + 名称 + 文件数 + 创建者。

#### 5.2.2 进入空间

- 点击文件进入详情 → 互动；

#### 5.2.3 无管理入口

### 5.3 Tab3 上传（中央按钮）

#### 5.3.1 入口

底部导航中央放大按钮；点击弹 ActionSheet：拍照 / 从相册 / 取消。

#### 5.3.2 完整流程

1. 选择文件（≤ 50）；
2. 预览 + 排序 + 删除；
3. **选择目标群组**（列表展示「我的群组」+ 所有受邀群组，可多选）；

#### 5.3.4 异常

- token 失败：提示重试；

## 7. 权限矩阵与数据隔离规则

### 7.1 群组级隔离

所有文件、分类、空间、互动 SQL 层强制 `group_id` 过滤。

### 7.2 空间级权限

- `visibility ∈ {public_in_group, members, private}`；
- `public_in_group`：群组内全员可见；
- `members`：`space_members` 表中的用户可见；
- `private`：仅创建者可见。

### 7.3 后台管理权限

- 路由级权限守卫：根据角色返回不同的导航菜单和数据范围；
- 所有用户：影像上传 + 我的空间；
- 主理人额外：群组管理 + 影像管理 + 空间管理 + 互动管理 + 素材主题管理；
- 超管额外：用户管理 + 系统设置 + 全局素材。

## 8. 核心业务流程

### 8.1 注册 & 初始化

1. 获取验证码 → 填写用户名/密码/验证码 → 提交；
2. 后端校验 → 写入 users → 建「我的群组」+ 默认分类 + 默认首页皮肤（科技风「银河漫游」）；
3. 返回 JWT。

### 8.2 上传流程（APP + Admin 共享）

```
[选择文件] → [选择目标群组（可多选）] → [选分类 + 填描述]
  → [对每个群组分别申请 upload token] → [直传七牛]
  → [七牛回调入库（每个群组各一条 file 记录）]
  → [时光轴可见]
```

**适用端**：APP Tab3 上传 + Admin 后台「影像上传」模块（同一 API）。

文件状态机：`draft → uploading → pending_callback → ready → failed`

### 8.3 空间策展流程（后台）

```
[用户登录后台] → [我的空间 → 创建空间]
  → [填名称/描述/权限] → [进入策展页面]
  → [从群组内容筛选文件] → [加入空间 + 排序]
  → [配置空间皮肤] → [保存]
  → [APP 端空间列表可见]
```

### 8.4 首页皮肤配置与应用流程

**后台配置（超管/主理人）：**

```
[素材主题管理 → 创建风格] → [在风格下创建主题]
  → [上传背景图/视频 + 配置动效参数 JSON + 上传 BGM + 选择字体 + 设置色彩模式]
  → [预览 → 上架]
```

**APP 应用（所有用户）：**

```
[时光页右上角「皮肤」] → [弹出面板 → 浏览风格/主题]
  → [点击预览] → [确认应用]
  → [写入 user_active_theme] → [首页热切换（背景/动效/BGM/字体）]
```

### 8.5 群组切换

- 切换 → 原生效 false + 新生效 true → 清缓存 → 重载。

### 8.6 成员添加

```
[主理人后台] → [群组管理 → 添加成员]
  → [搜索用户名] → [选择] → [确认]
  → [写入 group_members] → [对方消息通知]
```

### 8.7 收藏权限联动

- 空间权限变更 → 识别受影响用户 → 写消息 + 收藏 `hidden=true/false`。

### 8.8 自定义素材审核

- 草稿 → 提交 → 待审核 → 通过/驳回。

## 10. 接口清单
